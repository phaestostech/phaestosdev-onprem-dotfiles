# Secrets management

All secrets are stored in the **phaestosdev-onprem** 1Password vault and injected at apply time using chezmoi's `onepasswordRead` function.

## How it works

Chezmoi calls the 1Password CLI (`op`) to read secrets during `chezmoi apply`. Templates reference secrets using the `op://` URI format:

```go
{{ onepasswordRead "op://vault-name/item-name/field-name" }}
```

The 1Password CLI must be authenticated before running `chezmoi apply`:

```bash
eval $(op signin)
```

## Add a secret

1. Store the secret in the **phaestosdev-onprem** vault in 1Password. Use the item name as a readable label and store the value in the `credential` field.
2. Reference it in your template:
   ```go
   {{ onepasswordRead "op://phaestosdev-onprem/My Item Name/credential" }}
   ```
3. Test locally:
   ```bash
   chezmoi diff
   chezmoi apply
   ```

## Add a per-machine secret

For secrets that differ per machine (like the Obsidian API key), use `promptStringOnce` in `.chezmoi.toml.tmpl`:

```go
{{- $mySecret := promptStringOnce . "mySecret" "Prompt text" "default-value" -}}
```

Then reference `{{ .mySecret }}` in any template. Chezmoi prompts once and stores the value in `~/.config/chezmoi/chezmoi.toml`.

## Security notes

- Rendered secrets exist only in target files (e.g. `~/.zshrc`), never in the source repo
- The `private_` prefix ensures SSH configs get `0600` permissions
- `~/.config/chezmoi/chezmoi.toml` contains `promptStringOnce` values -- treat this file as sensitive
- The `.chezmoiignore` file excludes credential files like `.claude/.credentials.json` and `.codex/auth.json`
